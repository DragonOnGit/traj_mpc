# Traj MPC Package

基于ROS1和MAVROS的无人机MPC轨迹跟踪控制功能包。

## 功能特点

1. **MPC参数可调节**：通过launch文件或ROS参数服务器调整MPC控制器参数
2. **话题可配置**：系统状态订阅话题可通过launch文件修改，默认为/Odometry
3. **速度控制**：控制指令使用速度控制，通过/cmd_vel话题发布
4. **XML轨迹定义**：期望轨迹由XML文件定义，关键点使用point标签表示
5. **四元数姿态**：轨迹点包含位置和姿态四元数信息

## MPC参数调节思路

### 1. 预测 horizon (horizon)
- **作用**：定义MPC预测的时间步数
- **调节建议**：
  - 较小的值（5-10）：响应速度快，计算量小，但跟踪精度可能降低
  - 较大的值（10-20）：跟踪精度高，但计算量增加，可能导致实时性问题
  - **初始值**：10

### 2. 时间步长 (dt)
- **作用**：定义MPC的时间步长
- **调节建议**：
  - 应与控制频率匹配，通常为1/control_rate
  - 较小的值：控制更精细，但计算量增加
  - 较大的值：计算量减少，但控制精度可能降低
  - **初始值**：0.1（对应10Hz控制频率）

### 3. 权重参数

#### 位置权重 (weight_pos)
- **作用**：控制位置跟踪精度的权重
- **调节建议**：
  - 较大的值：优先保证位置跟踪精度
  - 较小的值：允许较大的位置误差，可能使系统更稳定
  - **初始值**：10.0

#### 速度权重 (weight_vel)
- **作用**：控制速度平滑性的权重
- **调节建议**：
  - 较大的值：速度变化更平滑
  - 较小的值：速度响应更快速
  - **初始值**：1.0

#### 加速度权重 (weight_acc)
- **作用**：控制加速度平滑性的权重
- **调节建议**：
  - 较大的值：加速度变化更平滑，系统更稳定
  - 较小的值：加速度响应更快速，可能导致系统振荡
  - **初始值**：0.1

### 4. 约束参数

#### 最大速度 (max_vel)
- **作用**：限制无人机的最大线速度
- **调节建议**：
  - 根据无人机的实际性能设置
  - 较大的值：允许更快的运动，但可能导致不稳定
  - 较小的值：运动更保守，更稳定
  - **初始值**：2.0 m/s

#### 最大加速度 (max_acc)
- **作用**：限制无人机的最大加速度
- **调节建议**：
  - 根据无人机的实际性能设置
  - 较大的值：允许更剧烈的运动
  - 较小的值：运动更平滑，更稳定
  - **初始值**：1.0 m/s²

### 5. 控制频率 (control_rate)
- **作用**：MPC控制器的更新频率
- **调节建议**：
  - 较高的频率：控制更及时，跟踪精度更高
  - 较低的频率：计算量减少，但跟踪精度可能降低
  - **初始值**：10.0 Hz

## 实际测试步骤

1. **空载测试**：在安全环境下，先进行空载测试，观察无人机对轨迹的跟踪情况
2. **参数微调**：根据实际跟踪效果，逐步调整MPC参数
3. **负载测试**：在不同负载情况下测试系统性能
4. **复杂轨迹测试**：使用更复杂的轨迹测试系统的鲁棒性
5. **室外测试**：在实际室外环境中测试系统性能

## 轨迹文件格式

轨迹文件使用XML格式，示例如下：

```xml
<?xml version="1.0"?>
<trajectory>
  <point x="0.0" y="0.0" z="1.0" qx="0.0" qy="0.0" qz="0.0" qw="1.0" />
  <point x="1.0" y="0.0" z="1.0" qx="0.0" qy="0.0" qz="0.0" qw="1.0" />
  <!-- 更多关键点 -->
</trajectory>
```

每个point标签包含七个属性：
- x, y, z: 位置坐标
- qx, qy, qz, qw: 姿态四元数

## 注意事项

1. 确保MAVROS已正确安装和配置
2. 确保无人机的状态话题正确发布
3. 初次测试时，建议使用保守的参数设置
4. 在室外测试时，注意安全，确保周围环境空旷
5. 根据无人机的实际性能调整参数
