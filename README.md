# Traj MPC Package

基于ROS1的MPC轨迹跟踪控制功能包（无MAVROS依赖）。

## 功能特点

1. **MPC参数可调节**：通过launch文件或ROS参数服务器调整MPC控制器参数
2. **话题可配置**：系统状态订阅话题可通过launch文件修改，默认为/odom
3. **速度控制**：控制指令使用速度控制，通过/cmd_vel话题发布
4. **XML轨迹定义**：期望轨迹由XML文件定义，关键点使用point标签表示
5. **四元数姿态**：轨迹点包含位置和姿态四元数信息
6. **无MAVROS依赖**：适用于各种机器人平台，不依赖特定的飞控系统
7. **参考轨迹发布**：将参考轨迹数据发布至/path_exp话题，便于可视化和调试
8. **速度限制**：支持配置最大速度和加速度参数，确保控制平滑性

## 程序执行流程

1. **初始化阶段**：
   - 加载MPC参数和话题配置
   - 加载XML轨迹文件
   - 初始化订阅器和发布器
   - 进入IDLE状态，等待里程计数据

2. **轨迹跟踪阶段**：
   - 接收到第一条里程计消息后，自动开始轨迹跟踪
   - 按顺序访问每个路标点
   - 计算当前位置与目标路标点的距离
   - 当距离小于阈值时，切换到下一个路标点
   - 发布目标位置和速度控制指令

3. **任务完成阶段**：
   - 所有路标点访问完成后，进入COMPLETED状态
   - 发布零速度指令停止运动
   - 节点继续运行，等待新的任务

## 启动方式

```bash
roslaunch traj_mpc traj_mpc.launch
```

## 话题配置参数

以下参数可以在launch文件中配置：

| 参数 | 默认值 | 描述 |
|------|--------|------|
| odom_topic | /odom | 里程计话题名称 |
| setpoint_pos_topic | /target_position | 目标位置话题名称（已弃用，使用速度控制） |
| cmd_vel_topic | /cmd_vel | 速度控制指令话题名称 |
| path_topic | /path_exp | 参考轨迹发布话题名称 |
| trajectory_file | trajectories/example_trajectory.xml | 轨迹文件路径 |

## 速度控制参数

| 参数 | 默认值 | 描述 |
|------|--------|------|
| max_velocity | 2.0 | 最大线速度 (m/s) |
| max_acceleration | 1.0 | 最大加速度 (m/s²) |

## 参考轨迹发布

- **话题名称**: `/path_exp`
- **消息类型**: `nav_msgs/Path`
- **发布频率**: 10Hz
- **数据格式**: 包含轨迹中所有路标点的PoseStamped消息数组，每个PoseStamped包含位置和姿态信息
- **坐标系**: map

## 控制模式

系统使用**速度控制模式**进行轨迹跟踪：

1. **速度控制**：直接发布速度指令到`/cmd_vel`话题
2. **位置控制**：已弃用，相关代码保留为注释
3. **MPC控制**：使用模型预测控制算法计算最优速度指令

## MPC参数调节思路

### 1. 预测 horizon (horizon)
- **作用**：定义MPC预测的时间步数
- **调节建议**：
  - 较小的值（5-10）：响应速度快，计算量小，但跟踪精度可能降低
  - 较大的值（10-20）：跟踪精度高，但计算量增加，可能导致实时性问题
  - **初始值**：10

### 2. 时间步长 (dt)
- **作用**：定义MPC的时间步长
- **调节建议**：
  - 应与控制频率匹配，通常为1/control_rate
  - 较小的值：控制更精细，但计算量增加
  - 较大的值：计算量减少，但控制精度可能降低
  - **初始值**：0.1（对应10Hz控制频率）

### 3. 权重参数

#### 位置权重 (weight_pos)
- **作用**：控制位置跟踪精度的权重
- **调节建议**：
  - 较大的值：优先保证位置跟踪精度
  - 较小的值：允许较大的位置误差，可能使系统更稳定
  - **初始值**：10.0

#### 速度权重 (weight_vel)
- **作用**：控制速度平滑性的权重
- **调节建议**：
  - 较大的值：速度变化更平滑
  - 较小的值：速度响应更快速
  - **初始值**：1.0

#### 加速度权重 (weight_acc)
- **作用**：控制加速度平滑性的权重
- **调节建议**：
  - 较大的值：加速度变化更平滑，系统更稳定
  - 较小的值：加速度响应更快速，可能导致系统振荡
  - **初始值**：0.1

### 4. 约束参数

#### 最大速度 (max_vel)
- **作用**：限制机器人的最大线速度
- **调节建议**：
  - 根据机器人的实际性能设置
  - 较大的值：允许更快的运动，但可能导致不稳定
  - 较小的值：运动更保守，更稳定
  - **初始值**：2.0 m/s

#### 最大加速度 (max_acc)
- **作用**：限制机器人的最大加速度
- **调节建议**：
  - 根据机器人的实际性能设置
  - 较大的值：允许更剧烈的运动
  - 较小的值：运动更平滑，更稳定
  - **初始值**：1.0 m/s²

### 5. 控制频率 (control_rate)
- **作用**：MPC控制器的更新频率
- **调节建议**：
  - 较高的频率：控制更及时，跟踪精度更高
  - 较低的频率：计算量减少，但跟踪精度可能降低
  - **初始值**：10.0 Hz

## 轨迹文件格式

轨迹文件使用XML格式，示例如下：

```xml
<?xml version="1.0"?>
<trajectory>
  <point x="0.0" y="0.0" z="1.0" qx="0.0" qy="0.0" qz="0.0" qw="1.0" />
  <point x="1.0" y="0.0" z="1.0" qx="0.0" qy="0.0" qz="0.0" qw="1.0" />
  <!-- 更多关键点 -->
</trajectory>
```

每个point标签包含七个属性：
- x, y, z: 位置坐标
- qx, qy, qz, qw: 姿态四元数
